Markdown 标记使用方式

标题

  • 文章的顶层标题使用二级标题（有时文章的标题并不需要写在文章里，比如大多数博客有专门的位置填写文章标题，此时这级标题可以省略）
  • 每个小节的标题使用三级标题
  • 小节中进一步分层组织时使用四级标题
  • 尽量少用五级标题和六级标题，考虑用有序列表和无序列表代替
  • 完全不用一级标题

斜体

在中文排版中不使用斜体。在英文排版中可用斜体表达强调，或表示书名、题目。

引用

以下情况使用引用标记：

  • 引述内容时（也可直接用引号替代）
  • 作示例时
  • 给出提示、警告等额外说明时

行内代码

某一行文字中嵌入简短代码时使用行内代码，如：

打开 Linux 虚拟终端，输入 echo 'Hello World' 。恭喜，你已经入门 Shell 了 ：）

代码块

展示多行代码时使用代码块，也可用于 XML、JSON、配置项等。尽量在使用代码块时给出语言标识，有些 Markdown 工具会针对该语言高亮显示其中的语言元素。如：

```java
public class Main {
public static void main(String[] args) {
System.out.println(“Hello World”);
}
```

图片

Markdown 中使用

来插入图片，这里的「图片名称」可以任取，但是推荐使用对图片主题具有描述性的文字。因为在一些网站或软件中，图片无法加载时会显示这个「图片名称」，这时至少还能给读者传递出一点有用信息。

转义符号

如果不想让 Markdown 标记生效，可以在标记的每个符号前加上反斜杠（\），这样这些符号将按原样显示，不再具有 Markdown 中的特殊意义。

不想让二级标题标记生效，可以使用 **\#\#**，将显示为 ##

这篇文章是在我搭建自己的博客后写下的第一篇文章，用来记录自己博客的开始

为什么要搭建自己的博客？

  • 记录自己学习生涯的点点滴滴
  • 类似费曼学习法的输出式练习
  • 构建一个笔记仓库，或者说是外脑

要记录些什么东西？

  • 编程
  • 绘画
  • 音乐
  • 阅读
  • ……

总的来说只要是让我有所感悟或一些学习经验我都会记录在上面

博客是用什么搭建的？

目前是用 node.js + hexo 搭建的，主题是 NexT 。毕竟这这样的静态博客部署起来还算方便，并且也不需要复杂的维护工作，况且我也没有对于一些额外功能的需求，所以暂且是先这样用着。以后可能打算通过搭建动态博客提高自己的实战能力，不过那也都是以后的事情了。整个博客的搭建过程有参考 codesheep 的博客搭建视频，在这里也说一声感谢。

一些补充和碎碎念

其实自己心里还是蛮忐忑的，毕竟一些笔记或学习记录只给自己看和放到网络上给他人的学习提供帮助和参考是完全两个概念，心里也会有所担心别人的指责或是不同的意见，并且也算是对自己要求的更加严格。不过从一方面也非常的欢迎若是有什么问题大家可以给予指正，毕竟互联网上大家交流学习起来也更加方便，可以一起提高相关能力。
或许有人会有疑问，为什么一个技术博客会有很多其他方面的内容，主要是我自己的兴趣爱好还算广泛，且对自己也有一定的要求，并不是纯粹的当作休闲娱乐的视角来看待这些东西，我自己也比较洗碗能够成为一名终身学习者。所以博客也不一定只能记录计算机相关的内容不是吗？当然以后会考虑把这些东西和编程相关文章分开，现在就姑且这样吧。
